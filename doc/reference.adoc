=== Queries

==== Value expressions

The `where` clause in a query is followed by a search condition.  This
condition is a Boolean expression consisting of operators and
operands.

===== Operators

The supported operators, in order of precedence, are:

[%header,width=75%,cols="5,2,5"]
|===
|Operator
|Associativity
|Description

|`filter()`
|
|filtering operator

|`=` `<>` `<` `>` `\<=` `>=` `like` `ilike`
|
|comparison operators

|`not`
|right
|logical negation

|`and`
|left
|logical conjunction

|`or`
|left
|logical disjunction
|===

Parentheses may be used to override these precedence rules.

===== Operands

Supported operand types are:

[%header,width=75%,cols="3,4,5"]
|===
|Operand type
|Example
|Description

|name
|`author`
|attribute name

|string literal
|`'Thomas à Kempis'`
|string constant

|number
|`33`
|numeric constant
|===

===== Attributes

Supported attributes are divided into two groups.  The first group are
provided by CCMS or the original data source and cannot be modified:

[%header,width=75%,cols="4,3,6"]
|===
|Attribute
|Type
|Description

|`id`
|`bigint`
|CCMS identifier

|`author`
|`text`
|`100 $a`

|`title`
|`text`
|`245 $a` and `245 $b`

|`full_vendor_name`
|`text`
|`999 13 $a`

|`availability`
|`text`
|`999 13 $z`
|===

The second group are shared within a project and may be modified using
the `update` command:

[%header,width=75%,cols="4,3,6"]
|===
|Attribute
|Type
|Description

|`decision`
|`boolean`
|true if the decision to take the project action has been made

|`fund`
|fund name
|selected from the project property `funds`
|===

=== Lexical structure

==== Constants

String constants are enclosed in single quotes, for example, `'It is a
beautiful day'`.

A single quote within a string constant is written as two single
quotes, for example, `'It''s a beautiful day'`.
